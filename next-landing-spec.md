## yuu-js 紹介ページ仕様書（Next.js 用）

### 1. ページの目的

- Discord ボット **yuu-js** の特徴・機能をわかりやすく紹介し、  
  「このボットを自分のサーバーに入れてみたい」と思わせること。
- 招待リンク・サポートサーバーへの導線を明確に配置すること。

### 2. 想定ユーザー

- 日本語圏の Discord サーバー管理者 / モデレーター
- 既に他のボットを使ったことがある、または導入を検討しているユーザー

### 3. 全体構成

ページはシングルページ構成。下記セクションを縦に並べる。

1. **Hero セクション**
2. **機能一覧セクション**
3. **スクリーンショット / UI イメージ**
4. **導入手順セクション**
5. **FAQ / よくある質問**
6. **フッター（リンク類）**

### 4. 各セクションの詳細

#### 4.1 Hero セクション

- 要素:
    - キャッチコピー:
        - 例: 「日本語コミュニティのための多機能 Discord ボット」
    - サブコピー:
        - 例: 「レベリング・経済・チケット・ロールパネル・管理コマンドをこれひとつで。」
    - CTA ボタン:
        - 「ボットを招待する」ボタン（招待 URL を `env` 経由で注入できるように）
        - 「サポートサーバー」ボタン（任意）
- デザイン要件:
    - モダンでフラット、ややポップな印象
    - ダーク/ライトどちらでも見やすい配色（好みでダーク寄り）
    - ボットのアイコンを丸く表示する領域を用意（実画像は後で差し替え）

#### 4.2 機能一覧セクション

- 見出し: 「主な機能」
- カードレイアウト（2〜3 カラム、レスポンシブ）
- カード例:
    - レベリング / ランキング
        - 説明: 「メッセージに応じて XP を付与し、/rank や /leaderboard でランキングを表示」
    - 経済システム
        - 説明: 「仮想通貨によるポイント管理（コマンド構成に合わせて調整可）」
    - チケットシステム
        - 説明: 「サポート用チャンネルをボタンひとつで自動作成」
    - ロールパネル
        - 説明: 「ボタンまたはセレクトメニューからロール付与」
    - 管理・モデレーション
        - 説明: 「/kick, /ban, /timeout, /purge などの管理コマンド」
    - サーバー & ユーザー情報
        - 説明: 「/server や /user で情報を簡単取得」

#### 4.3 スクリーンショット / UI イメージ

- 要素:
    - ロールパネルやチケットパネルなど、代表的なメッセージ UI のスクリーンショットを配置できる枠
    - 画像がまだ無い場合でも、ダミーカードやプレースホルダーでレイアウトが崩れないようにする
- 実装メモ:
    - 画像パスは仮で `public/screenshots/...` を想定
    - 将来的に Next/Image で最適化可能な作りにする

#### 4.4 導入手順セクション

- 見出し: 「導入手順」
- ステップ形式（1〜4 ステップ）
- 例:
    1. 「ボットを招待する」ボタンから、自分のサーバーに yuu-js を追加
    2. 管理者権限や必要な権限を確認
    3. `/config` コマンドで基本設定（サポートロールなど）を行う
    4. `/rolepanel` や `/ticketpanel` で各種パネルを作成

#### 4.5 FAQ / よくある質問

- よくある質問を 3〜5 個程度
- 例:
    - Q: 利用は無料ですか？
    - Q: 権限はどの程度必要ですか？
    - Q: 日本語以外の言語には対応していますか？
    - Q: バグ報告や要望はどこにすれば良いですか？

#### 4.6 フッター

- 著作権表記（例: `© 2026 yuu-js`）
- リンク:
    - GitHub リポジトリ
    - サポートサーバー
    - 利用規約 / プライバシーポリシー（あれば placeholder で OK）

### 5. 技術要件（Next.js 側）

- フレームワーク: **Next.js 14+（App Router）**
- 言語: TypeScript
- スタイリング:
    - Tailwind CSS または CSS Modules どちらでもよいが、**コンポーネント単位で再利用しやすい設計**にする
- レイアウト:
    - `app/layout.tsx` で共通 `<html>` / `<body>` とフォント・テーマ設定を行う
    - メインページは `app/page.tsx` として実装
- メタデータ:
    - タイトル: `yuu-js | Discord 多機能ボット`
    - 説明: `日本語コミュニティ向けのレベリング・チケット・ロールパネル対応 Discord ボット`
    - OGP 用の画像パス（仮）を設定

### 6. コンポーネント分割方針

- `components/Hero.tsx`
- `components/Features.tsx`
- `components/Screenshots.tsx`
- `components/Steps.tsx`
- `components/FAQ.tsx`
- `components/Footer.tsx`

それぞれのコンポーネントは props でテキストを差し替えやすくしておくと、将来的な i18n / 文言変更がしやすくなる。

### 7. 文言・リンクの差し替え方針

- 文言や URL は、できるだけ **1 箇所の設定オブジェクト**から参照する。
    - 例: `config/site.ts` に `siteName`, `inviteUrl`, `supportServerUrl` などを定義し、各コンポーネントから import する。
- これにより、将来的に Bot 名や URL が変わってもコード修正範囲を最小限に抑えられる。
